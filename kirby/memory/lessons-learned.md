## 1. 3D 렌더링 이미지에서 픽셀 색상 감지는 신뢰할 수 없다 (2026-03-26, login-page)

**사건**: 3D 와이어프레임 R 로고 이미지에서 구리색 픽셀을 감지하여 노드 배치 좌표를 추출하려 했으나, 그림자/반사/조명에 의해 왼쪽 스트로크가 감지되지 않아 형태가 반복적으로 틀어짐
**교훈**: 픽셀 색상 기반 형태 감지 시, 3D 렌더링 이미지 대신 **플랫한 2D 단색 이미지**를 소스로 사용해야 안정적
**적용법**: 이미지 기반 형태 추출이 필요할 때 → 항상 균일한 단색 실루엣 이미지를 소스로 사용

## 2. 로컬 HTML에서 Canvas getImageData는 CORS로 차단된다 (2026-03-26, login-page)

**사건**: `r_logo.png`를 `<img>` → Canvas → `getImageData()`로 읽으려 했으나, 로컬 파일 CORS 정책으로 tainted canvas 에러 발생
**교훈**: 로컬 HTML 프로토타입에서 이미지 픽셀을 읽으려면 **base64 데이터 URL로 임베드**해야 함
**적용법**: HTML 프로토타입에서 이미지 처리 필요 시 → sips로 축소 → base64 임베드

## 3. 랜덤 샘플링은 재현 불가능한 결과를 만든다 (2026-03-26, login-page)

**사건**: Math.random() 기반 포아송 디스크 샘플링으로 노드 배치 → 새로고침할 때마다 다른 형태 → 좋은 결과를 재현할 수 없음
**교훈**: 시각적 결과의 재현성이 중요할 때 **시드 고정 PRNG**(mulberry32 등)를 사용
**적용법**: 디자인 프로토타입에서 랜덤 요소 사용 시 → 시드 고정으로 일관성 확보, 시드 변경으로 변형 탐색

## 4. 배경 색상은 참고 사이트를 직접 분석해서 맞추는 것이 효과적이다 (2026-03-26, login-page)

**사건**: WebGL 셰이더로 복잡한 배경을 만들었으나 색상이 맞지 않음 → 참고 사이트(alibaba page-agent)를 분석한 결과 단순 정적 CSS 그라데이션이었음
**교훈**: 배경 디자인 시 과도한 기술보다 **참고 사이트의 실제 구현을 먼저 분석**하고, 필요한 만큼만 구현
**적용법**: 참고 디자인이 있을 때 → WebFetch로 실제 CSS/JS 분석 → 최소한의 기술로 동일한 느낌 구현
